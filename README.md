DSCI 6004 – Natural Language Processing | University of New Haven, MS Data Science, Spring 2026 
Authors: Alima Aqsai & Ellen Martin


Overview

This project implements a Retrieval-Augmented Generation (RAG) pipeline to support psychiatric diagnosis and treatment planning. Instead of relying purely on 
what a language model learned during training, the system retrieves relevant passages from a curated library of clinical documents and uses 
those passages to generate grounded, source-backed answers to psychiatric questions.

Three open-source models were compared within the same pipeline:
- Mistral-7B-Instruct-v0.1
- LLaMA-3.1-8B-Instruct
- Gemma 2-9B-IT

Pipeline

```
PDF Documents
     │
     ▼
PyMuPDFLoader  ──►  RecursiveCharacterTextSplitter  ──►  Chunks (1000 chars, 100 overlap)
                                                               │
                                                               ▼
                                                     pubmedBERT Embeddings
                                                               │
                                                               ▼
                                                          ChromaDB
                                                               │
                                          ┌────────────────────┘
                                          │
                                     Query → Embed → Retrieve top-k (k=3)
                                          │
                                          ▼
                              RetrievalQA Chain ("stuff" mode)
                                          │
                                          ▼
                              Mistral / LLaMA / Gemma  ──►  Answer + Sources
```

Evaluation

Performance was assessed across four dimensions:

| Metric | Category | What it measures |
|---|---|---|
| BLEU | Lexical | Exact n-gram precision against retrieved reference text |
| ROUGE-L | Lexical | Longest common subsequence recall |
| roBERTScore (P/R/F1) | Semantic | Token-level semantic similarity via RoBERTa embeddings |
| Precision@k | Retrieval | Proportion of top-k chunks from the correct gold source |
| Recall@k | Retrieval | Whether the gold source appears in top-k results |
| MRR | Retrieval | Mean reciprocal rank of first correct retrieved document |
| Reference Relevance | Human | Author rating (0–2): grounded in retrieved material? |
| Clinical Accuracy | Human | Author rating (0–2): clinically correct and free of errors? |

---

Key Results

| Model | BLEU | ROUGE-L | BERTScore F1 |
|---|---|---|---|
| Mistral-7B | 0.085 | 0.244 | 0.860 |
| LLaMA-3.1-8B | ~0.03 | ~0.21 | 0.848 |
| Gemma 2-9B | ~0.085 | ~0.24 | 0.845 |

- ROUGE-L was significantly higher than BLEU for all models (paired t-test, p < 0.0001), indicating that models captured clinical concepts but paraphrased rather than quoted sources directly.
- Retrieval recall plateaued at k=3 (Recall@k = 0.764). Increasing k beyond this added noise.
- MRR stabilised around 0.6 across k values.
- Mistral performed best on most automated metrics; Gemma was the only model to explicitly flag when source material did not cover a question.

---

Limitations

- Small document base — five documents leave gaps for questions on AI in psychiatry and complex comorbidity scenarios.
- Chunk size mismatch — pubmedBERT's 512-token limit means some 1,000-character chunks may be silently truncated during embedding.
- BERTScore truncation — answers were truncated to 120 words before BERTScore computation due to GPU constraints.
